## Context

You are maintaining a living intelligence document for Ian's portfolio website. This document captures all relevant information needed to keep the About, Services, and other sections accurate and up-to-date.

### Source files to scan

- Project data: !`cat data/projects.ts`
- Project types: !`cat types/project.ts`
- About section: !`cat components/sections/about.tsx`
- Services section: !`cat components/sections/services.tsx`
- Hero section: !`cat components/ui/shape-landing-hero.tsx`
- Existing intel (if any): !`cat .tmp/portfolio-intel.md 2>/dev/null || echo "No existing intel file found."`

## Your Task

Analyze ALL the source files above and extract every piece of relevant information. Then write (or update) the file `.tmp/portfolio-intel.md` with the following structure. Be thorough — capture everything, not just what's obvious.

### Rules

1. **Do NOT invent information.** Only record what is explicitly present in the source files or what the user has stated in conversation.
2. **Deduplicate.** If a technology appears in multiple projects, list it once but note which projects use it.
3. **Preserve existing entries.** If `.tmp/portfolio-intel.md` already exists, merge new information into it rather than overwriting. Never remove an entry unless it was corrected by the user.
4. **Flag gaps.** If a section is missing information (e.g., no live demo URLs, no project images), note it with a `<!-- NEEDS: description -->` comment so we know what to fill in later.
5. **Keep it structured.** Use the exact format below so other skills and processes can parse it.
